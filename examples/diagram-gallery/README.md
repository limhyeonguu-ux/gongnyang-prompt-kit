# 복잡 도표 갤러리 — C6 돌파 전술 실측 (2026-07)

gpt-image-2(Codex `$imagegen`)로 복잡한 도표·개념 설명 이미지를 뽑은 40컷 실측 모음.
"모델은 도표를 못 그린다"는 통념을 반증하고, **어디까지 되고 무엇이 레버인지**를 기록한다.

> ⚠️ **품질 안내**: 이 갤러리 컷은 실측 그대로다. **중간중간 뭉개진 요소·소규모 한글 오탈자가 섞여 있을 수 있다** —
> 특히 텍스트 밀도가 400자를 넘는 슬라이드에서 간헐적으로 발생한다. 이는 결함이 아니라 **고밀도 산출의 정상 특성**이다.
> 치명 카피(제목·핵심 수치)만 따옴표로 고정해 지키고, 자유 작성 존의 본문은 소규모 오탈자를 허용한 뒤
> 걸리는 컷만 재생성하는 것이 운용 방식이다. 생성 이미지 위에 글자를 코드로 덧씌우는 후처리 합성은 하지 않는다(스킬 철칙 9).

각 폴더의 `prompts-*.jsonl`이 그 컷들을 만든 원본 프롬프트다. `codex_imagegen_runner.py`가 배치 러너(레이스-세이프 회수).

## 실측으로 굳어진 것

1. **텍스트 정확도의 1차 레버는 캔버스 세로 높이.** 21:9(codex 실측 세로 ~810px)에선 400자에서 글자가 작아져 뭉개진다.
   16:9(~950px)·2:3로 세로를 확보하면 700~800자도 또렷해진다. codex 경로는 큰 캔버스 요청(3440×1440 등)을 무시하고
   **~1900px 장변으로 정규화**하므로, 정확도는 비율 선택으로 벌어야 한다.
2. **자유 작성 존이 밀도의 핵심 무기.** 크리티컬 라벨만 따옴표 고정하고 본문은
   `genuine Korean sentences, fully written in real hangul`로 위임하면, 모델이 개념적으로 맞는 설명을 스스로 채운다
   (B-트리 키값 분배·Raft 로그 인덱스·캐시 트레이드오프까지 논리 정합).
3. **시퀀스·다대다도 정면 돌파된다.** TCP/OAuth 시퀀스 다이어그램(라이프라인+교차 메시지),
   트랜스포머 어텐션 팬(다대다), 블록체인 노드 상호검증(교차선+브리지 갭)이 방향·순서까지 맞았다.
   관건은 모델 능력이 아니라 **노드·연결·방향을 문장으로 얼마나 구체적으로 박느냐**다.
4. **완벽주의를 버리면 밀도가 열린다.** 소규모 오탈자·간헐 뭉개짐을 허용하면 슬라이드당 20~30개 텍스트 블록을 채울 수 있다.

전술 상세는 스킬 본체 `skills/image-prompt/references/category-patterns.md` C6 절 참고.
