# 타이포그래피 포스터 — TP1~TP14 라우터

> **글자 자체가 이미지인 포스터** 문법. 홍보판촉물(P1~P8)이 "제품·브랜드를 디자인된 물건으로 보이게" 한다면, 이 레이어는 **타이포그래피가 유일한 주인공인 포스터 아트** — 보는 사람이 "이게 글자로 만든 거라고?" 하게 만드는 14종.
>
> **쓰는 법: 아래 표에서 패턴 1개를 고르고 `references/typo-poster/<파일>` 하나만 읽어 드롭인 블록을 가져온다.** 14개를 다 읽지 않는다.

## 공통 DNA (전 패턴 공유)

① **단어가 주인공** — 렌더 단어 1개(또는 2~3어 구)가 화면 최대 시각 요소. 일시·장소·크레딧은 소형 위성 텍스트로 위계 분리(롤 라벨, 한 줄 한 언어). ② 초대형 렌더 타이포 상시 → **기본 Tier-1**(결합 공식 1회). 단 **반복·미세 텍스트가 컨셉인 TP2·TP14는 결합 공식과 모순**(`no duplicate text`가 반복 디자인을 죽임) — 화이트리스트 서브셋 `no invented glyphs, no watermark`만 쓴다. ③ 팔레트 **2~4색 하드 락** + HEX. ④ 여백 통제(TP6·TP7)든 밀도 폭발(TP11)이든 패턴이 정한 극단을 지킨다 — 중간값이 가장 나쁘다. ⑤ 파괴·왜곡·블러가 있어도 **자형 판독이 성립하는 지점**을 프롬프트에 명시한다(각 파일의 주의 참조).

## 경계·교차 참조

- **TP1 ↔ P1 타이포-마스크**: P1은 홍보물(페일 필드+메타 행+바코드), TP1은 포스터 풀블리드 스택. 요청이 "포스터·여행·도시"면 TP1, "매거진·홍보물"이면 P1.
- **TP3 ↔ P2 타이포-환경**: P2는 인물 서사(실루엣이 글자 위를 이동), TP3는 글자 자체가 건축물(인물 없이 성립).
- **TP4 ↔ C11 `shadow_narrative`**: C11은 그림자가 장면의 서사, TP4는 그림자·반사가 **글자를 렌더**.
- **TP5 ↔ P6 스트리트 콜라주**: P6은 스크랩 밀도의 화면, TP5는 파괴된 글자 단독 주인공.
- **TP11 ↔ L프리셋**: 애시드는 팔레트가 곧 정체성 — L프리셋 HEX와 조합하지 않는다.

## 교배 규칙

- TP × M축(사조)·R축(몸 반응) 허용 — 기존 스윕 원칙대로 **축 1개만 변주**.
- TP × L룩은 질감·조명 토큰만(팔레트는 TP가 지정한 쪽 승리).
- **TP끼리 교배 금지**(한 컷의 서사 담당은 하나) — 유일한 예외: TP7의 재질을 TP2 터널·TP3 건축 표면에 입히는 **재질 이식**은 성립한다.

## 한글 크로스 공통 규칙

- 마스킹(TP1)은 **한글 2자 안전권**, 3D 압출·조각·풍선·크롬(TP3·7·8·9)은 **1~2자**.
- 블랙레터(TP12)는 한글 직역 불가 — 붓글씨·각진 궁서 어휘로 번역.
- 파괴·그런지(TP5)는 "자형은 또렷한 볼드 유지, 파괴는 가장자리와 표면으로만".
- 반복 랩핑·미세 텍스트(TP2·TP14)는 영문이 안전, 한글이면 2~4자 단어.

## 고위험 패턴 안내

**TP13 아나모픽**(시점 정합)·**TP14 미크로그래피**(미세 텍스트)는 첫 컷 실패가 정상 — 2~3회 리트라이 전제로 안내하고, 어중간한 정합 컷은 폐기한다(컨셉이 반쯤 성립하면 안 한 것보다 나쁘다). 임팩트는 이 둘이 최상급.
